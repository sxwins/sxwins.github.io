Algorithm 3.1  最適探索

選択基準：開始からの累積コスト g(s) のみを使用（予測値 ĥ(s) は不使用）

● Open List の先頭（g 値最小）のノード s を取り出す

● 隣接ノード追加後，g(s) 昇順に並び替える

1初期状態のコスト値を 0 としてオープンリストに追加する．クローズドリストを空に初期化する．
2while  オープンリストが空ではない． do
3オープンリストから先頭の要素 s を取り出す．クローズドリストに s を追加する（s を探査することに相当）．
4s が目標状態ならば，解は発見されたとして探索を終了．
5s から接続していてまだ探査していない状態をすべてオープンリストに追加する．オープンリスト内の状態の累積コスト g(s) を再計算し，累積コストが小さい順に並び替える．
6end while  探索を終了．
